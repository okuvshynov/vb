minbencoding
Minbencode is a binary encoding that defines a unique canonical representation — every distinct value has exactly one valid encoding, and the input must be exactly one such encoding. Your validator must reject any input that is not in canonical form.

Strings are length-prefixed base ten followed by a colon and the string, which is a raw byte sequence of exactly that length. For example 4:spam corresponds to 'spam'.
Lists are encoded as an 'l' followed by their elements (also minbencoded) followed by an 'e'. For example l4:spam4:eggse corresponds to ['spam', 'eggs'].
