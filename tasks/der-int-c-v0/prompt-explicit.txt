## DER INTEGER encoding

A DER-encoded INTEGER is a TLV (Tag-Length-Value) triplet:

### Tag

The tag byte must be `0x02` (INTEGER, primitive form).

### Length

The length field encodes the number of value (content) bytes that follow:

- **Short form** (1 byte): If the length is 0–127, it is encoded as a single byte with bit 7 clear. For example, length 3 is encoded as `0x03`.
- **Long form** (2–3 bytes): If the length is 128–65535, the first byte is `0x80 | n` where `n` is the number of subsequent length bytes (1 or 2), followed by `n` bytes encoding the length in big-endian with no leading zeros. For example, length 128 is `0x81 0x80`; length 256 is `0x82 0x01 0x00`.
- The **indefinite form** (`0x80`) is not allowed in DER.
- Short form must be used when possible — long form is only valid for lengths that cannot be represented in short form (i.e., 128 and above).
- Long-form length bytes must not have leading zeros.
- This validator only needs to support lengths representable in at most 2 subsequent bytes (max length = 65535). Reject any length field claiming more than 2 subsequent bytes.

### Value (content)

The value bytes encode a signed integer in big-endian two's complement. The content must be at least 1 byte (length 0 is invalid).

The encoding must be minimal: the value must use the fewest possible bytes. Because integers are encoded in two's complement, the most significant bit (bit 7) of the first content byte indicates the sign:

- **Positive values**: If the value is positive but the first content byte has bit 7 set, a leading `0x00` byte is required to keep the value positive. For example, 128 is encoded as `02 02 00 80` — without the `0x00`, the byte `0x80` alone would mean -128.
- **Negative values**: If the value is negative but the first content byte has bit 7 clear, a leading `0xFF` byte is required to keep the value negative. For example, -129 is encoded as `02 02 FF 7F` — without the `0xFF`, the byte `0x7F` alone would mean +127.
- **Redundant padding is forbidden**: A leading `0x00` byte is invalid if the next byte has bit 7 clear (value is already positive without it). A leading `0xFF` byte is invalid if the next byte has bit 7 set (value is already negative without it).

Examples:
- `02 01 00` — valid encoding of 0
- `02 01 7F` — valid encoding of 127
- `02 02 00 80` — valid encoding of 128 (0x00 pad needed)
- `02 01 80` — valid encoding of -128
- `02 02 FF 7F` — valid encoding of -129 (0xFF pad needed)
- `02 02 00 01` — invalid: redundant 0x00 (0x01 already has bit 7 clear)
- `02 02 FF FF` — invalid: redundant 0xFF (0xFF already has bit 7 set)
