## DER INTEGER encoding

A DER-encoded INTEGER is a TLV (Tag-Length-Value) triplet:

### Tag

The tag byte must be `0x02` (INTEGER, primitive form).

### Length

The length field encodes the number of value (content) bytes that follow:

- **Single-byte form**: If bit 7 is clear, the byte directly encodes the length (0–127). For example, length 3 is `0x03`.
- **Multi-byte form**: If bit 7 is set, the low 7 bits give the number of subsequent length bytes (`n`), which encode the length in big-endian. For example, length 128 is `0x81 0x80`; length 256 is `0x82 0x01 0x00`. A first byte of `0x80` (indefinite form) is not allowed.
- This validator only needs to support lengths up to 65535 (at most 2 subsequent length bytes). Reject any length field claiming more.

### Value (content)

The value bytes encode a signed integer in big-endian two's complement. The content must be at least 1 byte (length 0 is invalid).

DER requires minimal encoding: both the length field and the integer value must use the fewest possible bytes. Per X.690 §8.3.2, if the contents octets comprise more than one octet, then the bits of the first octet and bit 8 of the second octet shall not all be ones and shall not all be zero.
