## Specification

Your validator must accept a miniformat message as stdin.

If it is invalid, you need to exit with non-zero exit code.

miniformat encoding
Miniformat is a binary encoding that defines a unique canonical representation — every distinct value has exactly one valid encoding, and the input must be exactly one such encoding. Your validator must reject any input that is not in canonical form.

Strings are length-prefixed base ten followed by a '#' and the string, which is a raw byte sequence of exactly that length. For example 4#spam corresponds to 'spam'.
Sequences are encoded as an 's' followed by their elements (also miniformat-encoded) followed by a ';'. For example s4#spam4#eggs; corresponds to ['spam', 'eggs'].
